The Problem

Every SaaS company has the same bottleneck: feature requests. Customers want more all the time. They provide great feedback and ideas, but it's hard to build fast enough to keep up.

Even after 10x'ing development throughput with AI tools and doubling engineering bandwidth, users just keep wanting more. Solving problems elegantly begets more problems users want solved.

Many requests aren't complicated—just different arrangements of the same building blocks:

  • A custom dashboard using existing charts, organized for executives
  • Data export formatted a specific way for accounting
  • Bulk actions to archive thousands of records at once

The backend API can already do most of what they're asking for. The bottleneck isn't capability—it's exposing the capabilities to the user.

The Solution

n.codes is an open source project that lets users build the UI they need with just a prompt—inside your app. It's agentic UI, tightly integrated with your backend APIs, keeping a consistent style.

It's like a forward-deployed engineer for every user or Claude Code that lives in your app. Users ask what they need, and they get it right there, right now.

  1. Learn Components — Reads your frontend components to understand capabilities and styles.
  2. Build Capability Map — Reads the codebase, APIs, docs, and schemas to map entities, actions, queries, and constraints.
  3. Generate UI — Given a user prompt, builds the UI using the capability map. Runs in a sandboxed runtime.

Agent Rules

  • Safety — No arbitrary code execution. Permission-aware (RBAC scoped). Full audit trail.
  • Boundaries — For the 90%: CRUD, analytics, workflows. Not for marketing pages or bypassing business logic.
  • Ownership — Generated UIs can be private, shared with a team, or promoted to official by admins.

It's for the 80% of apps that are just different arrangements of the same building blocks.
